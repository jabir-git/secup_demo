## Best Practices

- Use type hints for clarity (PEP 484)
- Follow PEP 8 style guide consistently
- Write descriptive variable and function names
- Keep functions small and focused (single responsibility)
- Use list comprehensions and generators appropriately
- Handle exceptions explicitly, not silently
- Document complex logic, not obvious code
- Use meaningful variable names instead of comments
- Avoid mutable default arguments
- Use dataclasses or NamedTuple for structured data
